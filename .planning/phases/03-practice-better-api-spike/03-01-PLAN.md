  <action>
Create `scripts/pb-spike.ts` — a standalone Node.js script run with `npx tsx scripts/pb-spike.ts` that hits the live Practice Better API. This is NOT a test file — it makes real API calls.

The script must do the following steps in sequence, logging everything:

**Step 1: Token fetch**
- POST to `https://api.practicebetter.io/oauth2/token`
- Content-Type: `application/x-www-form-urlencoded`
- Body: `grant_type=client_credentials`, `client_id` from `process.env.PRACTICE_BETTER_CLIENT_ID`, `client_secret` from `process.env.PRACTICE_BETTER_CLIENT_SECRET`
- Log the FULL token response JSON (so we can see `access_token`, `expires_in`, `token_type`, and any other fields)
- If no `access_token` in response, log error suggesting the user try alternative field names (`api_key`/`api_secret`) and exit

**Step 2: Patient creation**
- POST to `https://api.practicebetter.io/consultant/records`
- Headers: `Authorization: Bearer {access_token}`, `Content-Type: application/json`
- Body: `{ "first_name": "StrengthRX", "last_name": "SpikeTest", "email": "USER_MUST_REPLACE@example.com" }`
- IMPORTANT: Add a comment above the email field telling the user to replace with a real email they control (to check for invite emails)
- Log the full response status AND full response body JSON
- If 4xx error, log the error body — it likely contains validation errors revealing required field names

**Step 3: Summary output**
- Print a formatted summary block at the end labeled `=== SPIKE FINDINGS ===` with:
  - Token: success/fail, `expires_in` value if present
  - Create: success/fail, response status code
  - ID field: which key in the response contains the patient ID (check `id`, `_id`, `client_id`, `record_id`)
  - Remind user to check their email for any Practice Better invite

The script must `import 'dotenv/config'` at the top to load `.env.local`.

Also update `.env.example` — append these lines at the bottom:

```
# Practice Better API (OAuth2 Client Credentials)
# Get these from Practice Better Settings > API
PRACTICE_BETTER_CLIENT_ID=your_api_key_here
PRACTICE_BETTER_CLIENT_SECRET=your_api_secret_here
```

Do NOT modify any existing lines in `.env.example`.
