**IMPORTANT: Read the 03-01-SUMMARY.md FIRST.** It contains the exact field names, response shape, and ID field discovered by the spike. Use those exact values — do NOT guess.

Create `src/lib/practice-better.ts` with:

1. **Constants:** `PB_BASE_URL = 'https://api.practicebetter.io'`

2. **Types (exported):**
   - `PracticeBetterClientInput` — `{ firstName: string, lastName: string, email: string, phone?: string }`
   - `PracticeBetterClientResult` — `{ id: string }`
   - `PracticeBetterError` — custom Error subclass with `status: number` and `body: string` properties

3. **Token cache (module-level, NOT exported):**
   - `cachedToken: string | null = null`
   - `tokenExpiresAt: number = 0`
   - `getAccessToken()` async function that:
     - Returns cached token if `Date.now() / 1000 < tokenExpiresAt`
     - Otherwise POSTs to `${PB_BASE_URL}/oauth2/token` with `Content-Type: application/x-www-form-urlencoded`
     - Body: `grant_type=client_credentials`, `client_id` from `process.env.PRACTICE_BETTER_CLIENT_ID`, `client_secret` from `process.env.PRACTICE_BETTER_CLIENT_SECRET`
     - On success: caches `data.access_token`, sets `tokenExpiresAt = now + (data.expires_in ? data.expires_in - 60 : 3300)`
     - On failure: throws `PracticeBetterError` with status and body

4. **`createPracticeBetterClient(input)` (exported):**
   - Calls `getAccessToken()`
   - POSTs to `${PB_BASE_URL}/consultant/records` with Bearer token
   - Request body: map `firstName` -> snake_case field name from spike findings, same for `lastName`, `email`, `phone`
   - **Use the exact field names from 03-01-SUMMARY.md** — the research hypothesizes `first_name`, `last_name`, `email` but the spike confirms the actual names
   - On success: extract patient ID using the field name confirmed by the spike (research suggests checking `id`, `_id`, `client_id`, `record_id`)
   - On failure: throw `PracticeBetterError` with status and response body
   - **Never log patient data** (PHI concern per research anti-patterns)

5. **Export a `resetTokenCache()` function** for testing purposes only — sets `cachedToken = null` and `tokenExpiresAt = 0`

Then create `tests/int/practice-better.int.spec.ts` with vitest tests using `vi.stubGlobal('fetch', mockFetch)` to mock the global fetch. Tests must cover all behaviors listed in the `<behavior>` block. Each test should call `resetTokenCache()` in beforeEach to ensure clean state.

Use `vi.spyOn(globalThis, 'fetch')` or `vi.stubGlobal` — check existing test patterns in the project first.

The env vars `PRACTICE_BETTER_CLIENT_ID` and `PRACTICE_BETTER_CLIENT_SECRET` should be stubbed with `vi.stubEnv` or set directly on `process.env` in beforeEach.
